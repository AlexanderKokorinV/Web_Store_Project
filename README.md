# Web_Store_Project

## Описание

Проект Web_Store_Project представляет собой веб-приложение на Django для управления каталогом товаров.

- С помощью Bootstrap 5 собраны прототипы страниц:
  - Главная (home.html)
  - Контакты (contacts.html)

  Соответствующие HTML-документы расположены в папке templates в корне проекта.

- Создано новое приложение под названием сatalog.
- Приложение зарегистрировано в настройках проекта.
- Настроена маршрутизация для нового приложения (добавлены соответствующие URL-адреса).
- Подключена СУБД PostgreSQL для работы в проект.
- Реализованы модели для Продуктов (Product), Категорий (Category) и Контактных данных (Contact).
- Перенесено отображение моделей в базу данных с помощью инструмента миграций.
- Создан суперпользователь, зарегистрированы модели Product и Category в админке, настроено отображение для моделей.
- Сформированы фикстуры для моделей Category и Product.
- Обратная связь: работающая форма на странице контактов с обработкой POST-запросов.

В проекте реализована кастомная команда для автоматизации управления данными:
- **python manage.py add_products** — специальная команда, которая:
    - Полностью очищает базу данных от старых категорий и продуктов.
    - Создает новые актуальные записи категорий и товаров из подготовленного списка.
    - Позволяет мгновенно подготовить проект к работе после установки.

**ВАЖНО:** в проекте используются чувствительные данные, такие как SECRET_KEY и др. Воспользуйтесь примером .env.example в корне проекта.


## Структура проекта


- `config/` — основная папка проекта с настройками:
  - `settings.py` — конфигурация проекта (подключение приложений, статики и БД).
  - `urls.py` — корневой конфигуратор маршрутов (URL).
- `catalog/` — основное приложение магазина:
  - `management/commands/` — кастомные команды Django (наполнение БД).
  - `migrations/` — история изменений базы данных.
  - `fixtures/` — данные для быстрого развертывания проекта.
  - `models.py` — модели `Product`, `Category`, `Contact`.
  - `templates/catalog/` — HTML-шаблоны страниц (home.html, contacts.html).
  - `screenshots/` — скриншоты выполнения команд Django shell.
  - `views.py` — контроллеры (обработка 5 последних товаров, вывод контактов).
  - `urls.py` — маршрутизация внутри приложения. 
  - `models.py` — описание структуры таблиц базы данных.
- `static/` — папка со статическими файлами (CSS/JS Bootstrap):
  - `css/bootstrap.min.css` — стили Bootstrap для оформления интерфейса.
- `manage.py` — утилита командной строки для управления проектом (запуск сервера, миграции).
 

## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/AlexanderKokorinV/Web_Store_Project
```
2. Установите зависимости:
```
poetry install
```

## Запуск программы

Запустите сервер:
```
python manage.py runserver
```
После установки и выполнения миграций (migrate), воспользуйтесь командой наполнения:
```
python manage.py add_products
```

## Тестирование
В проекте реализовано тестирование:
1. Тесты на доступность страниц (Status Code 200).
Проверка того, что страницы «Каталог» и «Контакты» открываются и не выдают ошибку 404 или 500.
2. Тест формы обратной связи.
Проверка того, что при отправке данных (POST-запрос) сервер отвечает успешно и выводит сообщение.

Для запуска тестов введите команду:
```
python manage.py test
```
